MatchGorithm

I work with a fellow university graduate to solve problems related to resource allocation. We have a mutually designed algorithm which efficiently solves various industry problems, and are trying to turn it into a practical application.

More specifically, the Matcher uses AI-adjacent concepts to process datasets until we get useful answers. It has configuration options to allow us to interact with every tiny aspect of it's inner workings, and can even do things such as stop early if it finds satisfactory answers. It includes a testing suite, various GUIs and an entire website to control access to the service.

Prototypes of our software were originally designed in Racket (a member of the Lisp family) to allow for more convenient data structure manipulation and automation. The current version uses JavaScript with a few Lisp-inspired coding concepts, and runs on node.js serverside.

Prayer Times Finder

A small application made to display the day's Islamic prayer times. Outputs directly to a terminal.

This application was made both as a way to test the usage of REST APIs, and the effectiveness of developing software on a phone. The vast majority of the code was written in Android's Termux terminal emulator, and was then neatened up on a computer after the fact. The application is it's own configuration file, with it's configuration struct appearing at the top of the file. This is to make it easier to work with on a tiny screen, so anyone making edits does not have to juggle multiple files.
